state는 리액트 컴포넌트 안에 있는 오브젝트이다. 
state가 바뀔 때 마다, 컴포넌트는 다시 render 할것이다.

다른 곳에서 state를 직접적으로 변경하면 안된다. ex)this.state
직접적으로 변경하면 위의 render 설정들이 작동을 안한다.
state를 업데이트 하려면 this.setState를 사용해야한다.
여기서 새로운 state를 만들어야한다.
state은 컴폰너트를 로드하는 방법이고, 디폴트 state와 함께
업데이트할 때마다 render이 작동할 것(새로운 state와 함께)

사람이 필요한 데이터가 항상 바로 즉시 존재하지는 않는다.
데이터 없이 컴포넌트가 로딩을 하고 데이터를 위해 api를 불러서 
api가 데이터를 주면 그 사람의 컴포넌트 state를 업데이트 한다.
api 콜을 타임아웃 기능으로 유사하게 구현해보기
